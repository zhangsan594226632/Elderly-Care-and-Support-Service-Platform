## 一、项目概述


### 1\.1 项目背景

当下老龄化程度持续加深，居家养老、社区养老需求激增，老年人日常健康管理、社区养老活动参与、上门养老服务预约、用药提醒、家属监护联动等需求缺乏线上一体化管理渠道。传统线下养老服务存在信息不互通、服务分配低效、老人健康档案零散、家属无法实时掌握老人动态等痛点。
基于此，本项目采用前后端分离架构，后端 SpringBoot、前端 Vue 开发**爱老助老服务平台**，区分普通用户（老人、监护人）、管理员两大操作端口，实现养老健康管理、社区活动、养老服务预约分配、健康科普、用药提醒全流程线上数字化管理。

### 1\.2 项目目标

1. 搭建双端访问体系：前台用户端（老人 / 监护人登录使用）、后台管理端（管理员统一运维）；

2. 实现老人、监护人、管理员、志愿者多角色账号注册、登录、权限隔离；

3. 完善老人健康档案管理，记录基础信息、慢性病、家族病史；

4. 社区养老活动发布、分类浏览、后台增删改查；

5. 养老服务线上预约、志愿者分配、服务记录留存；

6. 健康科普文章图文发布与阅览；

7. 用药、复诊提醒功能，监护人同步接收提醒信息；

8. 系统轮播图、基础数据统一后台管理。

### 1\.3 技术架构

#### 后端技术栈

- 核心框架：SpringBoot 2\.7\.x

- 持久层：MyBatis/MyBatis\-Plus

- 数据库：MySQL 8\.0

- 安全：SpringSecurity/Token 登录鉴权

- 文件上传：本地文件 / OSS 图片存储（头像、活动配图、科普图片）

- 工具：Lombok、Hutool、分页工具、日期工具

#### 前端技术栈

- 核心框架：Vue2/Vue3 \+ VueCli/Vite

- UI 组件库：Element UI / Element Plus

- 路由：VueRouter（前台、后台路由分离）

- 网络请求：Axios

- 富文本编辑器：WangEditor（活动内容、科普文章编辑）

- 页面功能：分页查询、条件检索、图片上传、下拉筛选、弹窗表单

#### 架构模式

前后端分离架构：

1. 前端 Vue 负责页面渲染、表单交互、数据展示；

2. SpringBoot 后端提供 RESTful API 接口；

3. 基于 Token 实现登录状态保持，区分用户角色权限；

4. MySQL 存储所有业务数据，图片资源独立存储。

## 二、系统功能整体设计

### 2\.1 用户角色划分

系统包含 4 类角色，权限完全隔离：

1. **老人用户**：注册账号，完善个人健康档案，查看活动、预约养老服务、查看用药提醒；

2. **监护人用户**：绑定老人账号，查看老人健康信息、接收老人用药提醒、协助预约服务；

3. **志愿者**：后台录入信息，接收管理员分配的养老服务工单；

4. **系统管理员**：后台全权限，管理所有用户、活动、服务、科普、提醒、轮播图数据。

### 2\.2 前台用户端功能模块（老人 / 监护人访问）

根据截图前台页面拆分 6 大模块：

1. **登录注册模块**

    - 登录页：账号、密码、用户类型下拉选择（老人 / 监护人）、记住密码；

    - 双注册入口：注册老人、注册监护人；

    - 老人注册表单：老人账号、密码、确认密码、姓名、头像上传、性别、年龄、手机号、绑定监护人账号；

2. **首页**：平台首页轮播图展示；

3. **健康科普模块**：分页浏览管理员发布的健康科普图文，支持查看详情；

4. **活动信息模块**

    - 活动搜索：按活动名称模糊检索；

    - 区域筛选：按活动区域标签快速筛选；

    - 活动卡片展示：活动封面、活动名称、活动地点；

5. **服务信息模块**：查看平台提供的各类养老服务；

6. **服务记录模块**：查看本人预约、已分配志愿者的历史服务工单；

7. **个人中心 \- 健康信息**：展示老人完整健康档案（姓名、年龄、慢性病、家族病史、监护人关联信息、登记时间）；

8. **提醒功能模块**：展示所有绑定老人的用药提醒，支持按日期区间检索、新增提醒。

### 2\.3 后台管理员端功能模块

管理员独立登录入口，左侧菜单栏全量运维功能：

1. **首页**：后台总览首页；

2. **轮播图管理**：前台首页轮播图新增、编辑、删除；

3. **系统用户管理**

    - 志愿者管理：志愿者账号信息维护；

    - 管理员管理：后台账号增删改查；

    - 老人管理：所有老人账号数据管理；

    - 监护人管理：监护人账号管理；

4. **健康科普管理**：科普文章列表、新增、修改、删除，支持图片上传、富文本内容编辑；

5. **活动信息管理**

    - 活动区域管理：维护活动分类区域；

    - 活动信息列表：分页展示所有活动，支持按名称搜索；新增弹窗包含活动名称、多图上传、活动时间、活动区域、地点、发布时间、富文本活动内容；支持详情、修改、删除操作；

6. **服务信息管理**：养老服务类型、服务基础信息维护；

7. **服务记录管理**

    - 服务分派：将预约订单分配给对应志愿者；

    - 服务记录：所有历史预约工单分页列表，展示服务名称、时间、老人、监护人、志愿者关联信息；

8. **健康信息管理**：统一管理所有老人健康档案；

9. **提醒功能管理**：全平台用药提醒统一管理，日期检索、新增提醒。

## 五、后端核心业务接口设计（RESTful）

### 5\.1 登录注册接口

1. `/api/login`：统一登录接口，接收账号、密码、角色类型，返回 Token；

2. `/api/older/register`：老人注册；

3. `/api/guardian/register`：监护人注册；

4. `/api/admin/register`：管理员注册。

### 5\.2 老人健康档案接口

1. `/api/older/get/{id}`：获取老人健康详情；

2. `/api/older/list`：管理员分页查询所有老人；

3. `/api/older/update`：更新老人健康信息。

### 5\.3 活动相关接口

1. `/api/region/list`：查询所有活动区域；

2. `/api/activity/page`：分页查询活动（支持名称、区域筛选）；

3. `/api/activity/add`：新增活动，支持多图片上传；

4. `/api/activity/update`：编辑活动；

5. `/api/activity/del`：删除活动；

6. `/api/activity/detail/{id}`：活动详情。

### 5\.4 健康科普接口

1. `/api/popular/page`：科普分页列表；

2. `/api/popular/add/update`：新增 / 修改科普图文；

3. `/api/popular/del`：删除科普。

### 5\.5 服务预约 \& 分派接口

1. `/api/service/page`：服务记录分页；

2. `/api/service/distribute`：管理员分配志愿者；

### 5\.6 提醒功能接口

1. `/api/remind/page`：按日期检索提醒；

2. `/api/remind/add`：新增用药提醒。

### 5\.7 文件上传接口

`/api/upload`：统一图片上传接口，返回图片访问地址（头像、活动图、科普图通用）。

## 六、前端核心模块实现说明

### 6\.1 路由规划

1. 前台路由组（无后台管理权限）

    - `/login`：前台登录

    - `/registerOlder`：老人注册

    - `/registerGuardian`：监护人注册

    - `/home`：首页

    - `/popular`：健康科普

    - `/activity`：活动列表

    - `/service`：服务信息

    - `/serviceRecord`：服务记录

    - `/healthInfo`：个人健康档案

    - `/remind`：用药提醒

2. 后台路由组（需管理员 Token 鉴权）

    - `/admin/login`：管理员登录

    - `/admin/index`：后台首页

    - `/admin/banner`：轮播图管理

    - `/admin/user/*`：四类用户管理

    - `/admin/popular`：科普管理

    - `/admin/activity`：活动管理

    - `/admin/service`：服务工单管理

    - `/admin/remind`：提醒管理

### 6\.2 公共组件封装

1. 分页组件：统一封装分页逻辑，所有列表页面复用；

2. 文件上传组件：多图上传弹窗，适配头像、活动配图；

3. 富文本组件：封装 WangEditor，活动、科普编辑页面复用；

4. 检索栏组件：日期区间、关键词搜索通用组件；

5. 弹窗表单：新增 / 编辑统一弹窗布局。

### 6\.3 登录鉴权逻辑

1. 登录成功后端返回 Token，前端存储至 localStorage；

2. Axios 请求拦截器自动携带 Token；

3. 响应拦截器捕获 Token 过期、未登录状态，自动跳转对应登录页；

4. 路由守卫区分前台 / 后台路由，无权限直接拦截跳转登录。

## 七、系统运行流程

1. **用户注册登录**

    - 老人 / 监护人进入前台登录页，无账号选择对应注册表单完成注册；输入账号密码、选择角色登录前台；

    - 管理员独立后台登录入口，登录后进入运维后台。

2. **前台用户操作流程**
监护人绑定老人账号后，可查看老人完整健康档案；浏览社区活动并预约养老服务；查看用药提醒；阅览健康科普知识。

3. **管理员后台运维流程**
管理员登录后，维护轮播图、管理所有老人 / 监护人 / 志愿者账号；发布社区活动、健康科普文章；处理养老预约工单、分配志愿者；统一管理全平台用药提醒数据。

## 八、项目特色与创新点

1. **双端口隔离**：普通养老用户前台、管理员运维后台完全分离，权限严格区分，数据安全可控；

2. **家属监护联动**：监护人绑定老人账号，同步查看健康、用药提醒，解决独居老人家属信息不对称问题；

3. **一站式养老服务闭环**：健康档案、社区活动、上门服务预约、用药提醒、健康科普一体化，覆盖居家养老全场景需求；

4. **可视化内容管理**：后台富文本 \+ 多图上传，可视化发布活动、科普内容，操作简单，无需代码；

5. **完善的数据运维能力**：所有业务数据支持分页、检索、新增 / 编辑 / 删除，满足社区养老平台日常运营管理。

## 九、部署说明

1. 后端 SpringBoot 打包为 Jar 包，配置 MySQL 数据库连接信息，服务器 Java 环境运行；

2. 前端 Vue 执行 npm run build 打包 dist 静态文件，部署至 Nginx；

3. Nginx 配置反向代理，前端静态资源转发、后端 API 请求转发；

4. 图片上传目录配置服务器持久化存储，保证图片访问；

5. 数据库初始化执行提供的 SQL 脚本，创建所有业务表、基础测试数据。

## 十、总结

本爱老助老服务平台基于 SpringBoot\+Vue 前后端分离架构，针对社区居家养老场景打造数字化管理系统。系统区分老人、监护人、管理员多角色，覆盖健康档案、社区活动、养老服务预约、志愿者分派、健康科普、用药提醒核心业务。前台面向老年用户操作简洁直观，后台提供完整的数据运维能力，解决传统线下养老服务信息分散、管理低效、家属监护断层的痛点，具备良好的实用价值与拓展性，可直接落地社区养老服务中心使用。
